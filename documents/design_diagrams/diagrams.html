4.1. Sequence Diagram: Создание заказа

sequenceDiagram actor Customer participant API as API (FastAPI) participant Service as OrderService participant DB as Database Customer->>API: POST /orders (маршрут, описание) API->>Service: Валидация и создание сущности Service->>DB: INSERT INTO orders DB-->>Service: Успех (id) Service->>Service: Генерация tracking_url Service->>DB: UPDATE tracking_url Service-->>API: Order DTO (данные заказа) API-->>Customer: 201 Created (JSON)

4.2. Sequence Diagram: Принятие заказа курьером

sequenceDiagram actor Courier participant API as API (FastAPI) participant Service as OrderService participant DB as Database Courier->>API: POST /orders/{id}/accept API->>Service: Проверка: статус CREATED? Service->>DB: SELECT status FROM orders WHERE id={id} DB-->>Service: status = CREATED Service->>DB: UPDATE orders SET courier_id={c_id}, status=ACCEPTED DB-->>Service: Успех Service-->>API: Обновленный Order DTO API-->>Courier: 200 OK

4.3. Sequence Diagram: Отслеживание заказа

sequenceDiagram actor User participant API as API (FastAPI) participant Service as TrackingService participant DB as Database User->>API: GET /track/{tracking_url} API->>Service: getOrderByTrackingUrl() Service->>DB: SELECT * FROM orders WHERE tracking_url={url} DB-->>Service: Данные (статус, геопозиция) Service-->>API: Tracking DTO API-->>User: 200 OK (статус, локация)
